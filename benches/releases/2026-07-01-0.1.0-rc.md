## Competitor Benchmark

The standard v1 single-file profile used a fixed 4 MiB piece length, private flag,
identical tracker, two warmups, ten randomized measured rounds, warm cache, and
seed `20260701`. Every ranked output passed independent payload and metainfo
validation and had info hash
`6fd397c6de29f77d0f0c1928e6c457240112204c`. Raw schema-versioned JSON, CSV,
commands, environment data, and summary are under
`benchmark-results/release-candidate-2026-07-01-standard/20260701T222121.436719Z/`.

| Tool | Median | Mean ± SD | Throughput | Peak RSS | Relative |
|---|---:|---:|---:|---:|---:|
| mkbrr 1.23.0 | 0.274 s | 0.276 ± 0.013 s | 13,886.0 MiB/s | 74.2 MiB | 1.00× |

BTPC is not the fastest tool in this profile; mkbrr leads. BTPC native used the
lowest peak RSS among the equivalent ranked tools and had a 0.93% timing CV.

After rebuilding the final artifacts, a second ten-round BTPC-only run produced
0.854 s native and 0.911 s Python medians with identical info hashes and peak RSS
of 23.1 MiB and 39.7 MiB. The native median moved by 4.66% from the full-matrix
run, within the 5% gate. Raw results are under
`benchmark-results/release-candidate-2026-07-01-final-btpc/20260701T223248.183960Z/`.

## Warnings And Limitations

- This source snapshot is not a Git checkout. `git` and `gh` cannot identify a
  repository, so GitHub CI and the release workflow artifact-testing matrix could
  not be remotely executed. Only the macOS arm64 host artifacts are frozen here.
- `torrenttools 0.6.2` injects an extra cross-seed field into the `info` dictionary.
  Its output is payload-valid but not equivalent to the primary profile, changes
  the info hash, and was correctly rejected as unranked by the independent
  validator. The corrected quick run is under
  `benchmark-results/release-candidate-2026-07-01-quick-fixed/`.
- Cargo warned that the private workspace manifests have no license metadata while
  building the sdist. This does not invalidate the local artifact, but release
  metadata must be resolved before publishing.
- Measurements are warm-cache, single-machine, single-file v1 results. They do not
  support broad fastest claims or competitor claims for directory, v2, hybrid, or
  cold-cache workloads.
- Criterion reported expected outliers and short-measurement warnings in selected
  kernels. Medians, dispersion, confidence intervals, and independent end-to-end
  reruns were retained; no warning corresponds to a correctness failure.
- Publishing credentials and production package registries were not exercised.
